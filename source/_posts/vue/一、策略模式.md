# 一、策略模式

该模式时开发过程中 最常见，最易使用的设计模式

策略模式指的是：定义一系列的算法，把它们一个个封装起来，将不变的部分和变化的部分隔开，实际就是将算法的使用和实现分离出来。

例如：

公司的年终奖是根据员工的工资和绩效来考核的，绩效为A的人，年终奖为工资的4倍，绩效为B的人，年终奖为工资的3倍，绩效为C的人，年终奖为工资的2倍

```
 var obj = {
    "A": function(salary) {
        return salary * 4;
    },
    "B" : function(salary) {
        return salary * 3;
    },
    "C" : function(salary) {
        return salary * 2;
    }
};
var calculateBouns =function(level,salary) {
    return obj[level](salary);
};
console.log(calculateBouns('A',10000)); // 40000
```

**优点**

可以有效地避免多重条件语句，将一系列方法封装起来也更直观，利于维护

**缺点**

往往策略集会比较多，我们需要事先就了解定义好所有的情况

# 二、代理模式

代理模式为对象提供与一个代用品或占位符，以便控制对它的访问。

控制对对象的访问。过滤掉不需要的请求叫“保护代理”。把一些开销很大的对象，延迟到真正需要它的时候才去创建的叫“虚拟代理”。“缓存”代理为一些开销大的运算结果提供暂时的存储。在下次运算时。如果传递进来的参数和之前的一致，则可以直接返回之前运算的结果。

### 示例一：虚拟代理

```
// 使用虚拟代理实现图片预加载
class MyImage{
    constructor(src) {
        this.img = document.createElement('img');
        this.img.src = src;
        document.body.appendChild(this.img);
    }

    setSrc(src) {
        this.img.src = src;
    }
}


// 使用代理加载图片
class ProxyImage{
    constructor(src) {
        this.img = new MyImage('img'); // 先加载本地的loding图，由于图片在本地所以可以直接显示
        this.img.img.onload = () => { // 加载实际图
            this.img.setSrc(src);
            this.img.img.onload = null;
        }
        this.img.img.onerror = () => { // 发生错误时加载错误占位图
            this.img.setSrc('https://fuss10.elemecdn.com/a/3f/3302e58f9a181d2509f3dc0fa68b0jpeg.jpeg')
            this.img.img.onerror = null
        }
    }
}
```

### 示例二：保护代理

PaaS低代码正是使用这种设计模式实现禁用对系统字段的设置值操作

```
// 使用保护代理实现对对象属性的权限管控

// 原对象 Person
const Person = {
  name: '张三',
  age: 22
}

// 保护对象
const Protect = {};
Object.keys(Person).forEach(key => {
    Object.defineProperty(Protect, key, {
        get: () => Person[key],
        set: (value) => {
            if(key === 'age') throw new Error('年龄不允许修改');
            Person[key] = value;
        }
    })
})

console.log(Protect.name);  // 张三
console.log(Protect.age);  // 22
Protect.name = '李四';  //李四
Protect.age = 18;  // Uncaught Error: 年龄不允许修改
```

### 示例三：缓存代理

```
// 使用缓存代理实现加速计算斐波那契数列

// 正常递归版本
function fibonacci (n) {
  if (n==0) {
    return 0;
  } else if (n==1) {
    return 1;
  } else {
    return fibonacci(n-1) + fibonacci(n-2);
  }
}

// 引入缓存代理
const cache = {}
function fibonacci (n) {
   if (n==0) {
     return 0;
   } else if (n==1) {
     return 1;
   } else {
        cache[n] = (cache[n-1] || fibonacci(n-1)) + (cache[n-2] || fibonacci(n-2));
        return cache[n];
    }
}
```

### 小结：

代理模式符合**单一职责原则**，其通过代理对象为原有对象添加新职责，同时也符合**开放封闭原则**，如果有一天不需要代理了直接请求原对象即可，不需要对对象进行任何的改动。
