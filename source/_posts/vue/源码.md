## 一、静态类型检查flow



vue源码中使用Flow作为它的静态类型检查工具，使用静态类型检查工具非常有利于大型项目的开发和维护。

Flow是facebook出品的js静态类型检查工具。它的工作类似typescript的工作，vue2.x没有使用ts，想要做类型检查，所以使用flow来做这部分工作。

Flow的类型检查语法和ts的很像，在文件头部增加如下注释即可开启文件类型检查



```
/*@flow*/
或者	
// @flow
....
```



根目录下有.flowconfig文件，这是flow的配置文件。

根目录下有个flow文件夹，这是flow的类型定义文件。源码中的自定义类型在这里能找到。



```
flow
├── compiler.js        # 编译相关
├── component.js       # 组件数据结构
├── global-api.js      # Global API 结构
├── modules.js         # 第三方库定义
├── options.js         # 选项相关
├── ssr.js             # 服务端渲染相关
├── vnode.js           # 虚拟 node 相关
```



对flow感兴趣的同学可以去[官网](https://flow.org/en/)了解更详细的信息。

#### vue版本



vue2.x最终渲染都是通过render函数，所以最终vue编译都会将代码编译成render函数的形式。

在我们用vue-cli初始化一个vue项目的时候，会询问我们使用哪个版本的vue，有Runtime-Only 版本和Runtime + Compiler两个版本选项。



- Runtime Only



我们使用webpack的话，已经使用了vue-loader，在运行前已经预编译好了，使用这个版本的话，里面只包含了运行时的vue代码，代码会更轻量，效率更高。



- Runtime + Compiler版本



如果我们代码中使用了如下的写法



```
new Vue({
  template: '<div>{{ hi }}</div>'
})
```



webpack的vue-loader并不会编译template字符串这部分代码，这时候就需要使用这个版本，里面包含了运行时的vue代码和编译的vue代码，文件会更大，效率会低一些。



为了降低性能损耗，觉得代码中不使用template的话，使用runtime-only版本的vue更好。

#### 生命周期钩子



从上面vue初始化定义的地方有执行initMixin()方法，这个方法定义的文件如下



```
// src/core/instance/init.js
export function initMixin (Vue: Class<Component>) {
    Vue.prototype._init = function (options?: Object) {
        //...
        initLifecycle(vm)
        initEvents(vm)
        initRender(vm)
        callHook(vm, 'beforeCreate')
        initInjections(vm)
        initState(vm)
        initProvide(vm)
        callHook(vm, 'created')
        // ...
    }

```



从上面可以看到往Vue原型上增加了_init方法，在new Vue()实例化的时候，会执行this._init(options)初始化。可以看到这时候会执行两个callHook，分别是beforeCreate、created两个钩子函数。



callHook定义在src/core/instance/lifecycle中,如下所示



```
export function callHook (vm: Component, hook: string) {
  const handlers = vm.$options[hook]
  if (handlers) {
    for (let i = 0, j = handlers.length; i < j; i++) {
      try {
        handlers[i].call(vm)
      } catch (e) {
        handleError(e, vm, `${hook} hook`)
      }
    }
  }
  if (vm._hasHookEvent) {
    vm.$emit('hook:' + hook)
  }
}
```



vm.$options中保存着我们通常写在组件中的data、computed、created、mounted等函数和对象。上面的init中callHook就会调用组件中的钩子。

可以去看到几点：



1. 在beforeCreate之后才执行initState()
2. inject的初始化在initState()之前，provide的初始化在initState()之后。说明先先接收到值，之后才会往下传递



#### nextTick的定义



js执行是单线程的，它是基于事件循环的，他只有一个主线程，通过异步模拟多线程工作，每个异步结果都是通过任务队列来调度。队列中存放着一个个task，主要分为两类，分别为macro task 和 micro task，并且每个 macro task 结束后，都要清空所有的 micro task。

在浏览器环境中，



- 常见的 macro task 有 setTimeout、MessageChannel、postMessage、setImmediate；
- 常见的 micro task 有 MutationObsever 和 Promise.then。



nextTick就是做这一块的整合。

nextTick的实现在src/core/util/next-tick.js中。



```
import { noop } from 'shared/util'
import { handleError } from './error'
import { isIOS, isNative } from './env'

const callbacks = []
let pending = false

function flushCallbacks () {
  pending = false
  const copies = callbacks.slice(0)
  callbacks.length = 0
  for (let i = 0; i < copies.length; i++) {
    copies[i]()
  }
}

let microTimerFunc
let macroTimerFunc
let useMacroTask = false

if (typeof setImmediate !== 'undefined' && isNative(setImmediate)) {
  macroTimerFunc = () => {
    setImmediate(flushCallbacks)
  }
} else if (typeof MessageChannel !== 'undefined' && (
  isNative(MessageChannel) ||
  // PhantomJS
  MessageChannel.toString() === '[object MessageChannelConstructor]'
)) {
  const channel = new MessageChannel()
  const port = channel.port2
  channel.port1.onmessage = flushCallbacks
  macroTimerFunc = () => {
    port.postMessage(1)
  }
} else {
  /* istanbul ignore next */
  macroTimerFunc = () => {
    setTimeout(flushCallbacks, 0)
  }
}

if (typeof Promise !== 'undefined' && isNative(Promise)) {
  const p = Promise.resolve()
  microTimerFunc = () => {
    p.then(flushCallbacks)
    if (isIOS) setTimeout(noop)
  }
} else {
  // fallback to macro
  microTimerFunc = macroTimerFunc
}

export function withMacroTask (fn: Function): Function {
  return fn._withTask || (fn._withTask = function () {
    useMacroTask = true
    const res = fn.apply(null, arguments)
    useMacroTask = false
    return res
  })
}

export function nextTick (cb?: Function, ctx?: Object) {
  let _resolve
  callbacks.push(() => {
    if (cb) {
      try {
        cb.call(ctx)
      } catch (e) {
        handleError(e, ctx, 'nextTick')
      }
    } else if (_resolve) {
      _resolve(ctx)
    }
  })
  if (!pending) {
    pending = true
    if (useMacroTask) {
      macroTimerFunc()
    } else {
      microTimerFunc()
    }
  }
  // $flow-disable-line
  if (!cb && typeof Promise !== 'undefined') {
    return new Promise(resolve => {
      _resolve = resolve
    })
  }
}
```



从上面代码可以看到，首先定义两个变量microTimerFunc、macroTimerFunc，分别对应micro task 的函数和 macro task 的函数。



- 对于macro task，按优先级依次检测setImmediate、MessageChannel是否支持，都不支持就使用setTimeout 0来实现。
- 对于micro task ，则检测是否支持Promise，不支持则指向macro task的实现，使用macro来代替



从上面flushCallbacks函数可以看到，会遍历callbacks，并且同步执行。说明nextTick会把一个tick内的任务，在一个同步任务内执行完，不会分多次异步去执行。

最以可以看到函数的最后提供了一个Promise化的调用，在不传cb的时候，可以使用promise的方式使用nextTick。



vue提供了两种调nextTick的方式，一种是全局的，上面initGlobalAPI中的定义Vue.nextTick。另一种是实例上的vm.$nextTick,定义在renderMixin()方法中，在vue定义的文件位置。两种方法最后调用的都是上面定义的nextTick方法。



#### 响应式的原理



vue2.x的响应式的核心主要使用了ES5的Object.defineProperty方法。



**Object.defineProperty()** 的作用是直接在一个对象上定义一个新属性，或者修改一个已经存在的属性



```
Object.defineProperty(obj, prop, desc)
1. obj 需要定义属性的当前对象
2. prop 当前需要定义的属性名
3. desc 属性描述符

栗子：
let a = {}
a.b = xxx

// 相当于下面
Object.defineProperty(a, b, {
    value: xxx,
    ...
})
```



这个方法里的desc属性描述符中包含了get和set以及其它的一些可选键值。



- get 相当于给对象的这个属性添加了getter方法，访问这个值的时候，会触发getter方法。
- set相当于给对象的这个属性添加了setter方法，修改这个属性的时候会触发这个方法



```
Object.defineProperty(a, b, {
    value: xxx,
    set () {...},
    get () {...}
    ...
})
```



这样就相当于将vue中的对象变成响应式了，每次改变都会触发一些方法。



#### 某些对象变化vue监听不到的解决方式



- 使用 Object.defineProperty 实现响应式的对象，当我们去给这个对象添加一个新的属性的时候，是不能够触发它的 setter 的。如下



```
var vm = new Vue({
  data:{
    a:1
  }
})
// vm.b 是非响应的
vm.b = 2
```



这时候vue提供了里的Vue.set的方法，

定义在src/core/observer/index.js中



```
export function set (target: Array<any> | Object, key: any, val: any): any {
...
  defineReactive(ob.value, key, val)
  ob.dep.notify()
  return val
}
```



可以看到最终调用了defineReactive,这个方法就是上面添加响应式对象的封装方法。最后notify再手动触发通知。



- 数组的也存在修改数组长度或者直接根据索引修改一个值不响应的情况。



在src/core/observer/array.js中



```
import { def } from '../util/index'

const arrayProto = Array.prototype
export const arrayMethods = Object.create(arrayProto)

;[
  'push',
  'pop',
  'shift',
  'unshift',
  'splice',
  'sort',
  'reverse'
]
.forEach(function (method) {
  // cache original method
  const original = arrayProto[method]
  def(arrayMethods, method, function mutator (...args) {
    const result = original.apply(this, args)
    const ob = this.__ob__
    let inserted
    switch (method) {
      case 'push':
      case 'unshift':
        inserted = args
        break
      case 'splice':
        inserted = args.slice(2)
        break
    }
    if (inserted) ob.observeArray(inserted)
    // notify change
    ob.dep.notify()
    return result
  })
})
```



可以看到vue对上述数组方法进行了重写，先执行原有的数组方法，然后加入新加的逻辑。新加的逻辑主要是将改变的数组项变成响应式，同时手动通知数据变化。



数组只要使用上述数组方法就可以出发响应变化



#### vm.$on、vm.$off、vm.$once、vm.$emit的实现



这几个api初始化是在初始化vue的时候eventsMixin(Vue)中，文件定义在src\core\instance\events.js中



##### vm.$on



```
const hookRE = /^hook:/
Vue.prototype.$on = function (event: string | Array<string>, fn: Function): Component {
const vm: Component = this
    if (Array.isArray(event)) {
      for (let i = 0, l = event.length; i < l; i++) {
        this.$on(event[i], fn)
      }
    } else {
      (vm._events[event] || (vm._events[event] = [])).push(fn)
      if (hookRE.test(event)) {
        vm._hasHookEvent = true
      }
    }
    return vm
}
```



从上面可以看到，事件名是个string的时候，会将定义的函数push进vm._events[event]中。事件名是个数组的时候会经过for循环递归调用自身，也会像上面string的方式处理。



##### vm.$off



```
Vue.prototype.$off = function (event?: string | Array<string>, fn?: Function): Component {
    const vm: Component = this
    // all
    if (!arguments.length) {
      vm._events = Object.create(null)
      return vm
    }
    // array of events
    if (Array.isArray(event)) {
      for (let i = 0, l = event.length; i < l; i++) {
        this.$off(event[i], fn)
      }
      return vm
    }
    // specific event
    const cbs = vm._events[event]
    if (!cbs) {
      return vm
    }
    if (arguments.length === 1) {
      vm._events[event] = null
      return vm
    }
    if (fn) {
      // specific handler
      let cb
      let i = cbs.length
      while (i--) {
        cb = cbs[i]
        if (cb === fn || cb.fn === fn) {
          cbs.splice(i, 1)
          break
        }
      }
    }
    return vm
  }
```



从上面可以看到，如果arguments.length === 1,则会执行vm._events[event] = null,相当于移除所有的event。如果提供fn，则会遍历event的数组去splice对应的event数组里的函数。如果传入的事件名是个数组，则会for循环递归调用自身，执行和string相同的操作。



##### vm.$once



```
Vue.prototype.$once = function (event: string, fn: Function): Component {
    const vm: Component = this
    function on () {
      vm.$off(event, on)
      fn.apply(vm, arguments)
    }
    on.fn = fn
    vm.$on(event, on)
    return vm
}
```



可以看到，使用$once添加事件的时候，会调用$on,并且执行事件的时候会调用$off，移除事件。



##### vm.$emit



```
Vue.prototype.$emit = function (event: string): Component {
    const vm: Component = this
    let cbs = vm._events[event]
    if (cbs) {
      cbs = cbs.length > 1 ? toArray(cbs) : cbs
      const args = toArray(arguments, 1)
      for (let i = 0, l = cbs.length; i < l; i++) {
        try {
          cbs[i].apply(vm, args)
        } catch (e) {
          handleError(e, vm, `event handler for "${event}"`)
        }
      }
    }
    return vm
  }
```



从上面可以看到，emit调用的时候，先会去vm._events[event]中查看有无定义这个事件，有的话会去遍历这个event的数组并且执行每一项的方法。从上面off可知，off之后的event会是一个空数组，这里for循环就不会走了。
