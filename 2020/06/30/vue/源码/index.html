vue源码中使用Flow作为它的静态类型检查工具，使用静态类型检查工具非常有利于大型项目的开发和维护。

二、vue文件分析

文件目录

vue版本

vue2.x最终渲染都是通过render函数，所以最终vue编译都会将代码编译成render函数的形式。

在我们用vue-cli初始化一个vue项目的时候，会询问我们使用哪个版本的vue，有Runtime-Only 版本和Runtime + Compiler两个版本选项。

  • Runtime Only

我们使用webpack的话，已经使用了vue-loader，在运行前已经预编译好了，使用这个版本的话，里面只包含了运行时的vue代码，代码会更轻量，效率更高。

  • Runtime + Compiler版本

如果我们代码中使用了如下的写法

webpack的vue-loader并不会编译template字符串这部分代码，这时候就需要使用这个版本，里面包含了运行时的vue代码和编译的vue代码，文件会更大，效率会低一些。

为了降低性能损耗，觉得代码中不使用template的话，使用runtime-only版本的vue更好。

nextTick的定义

js执行是单线程的，它是基于事件循环的，他只有一个主线程，通过异步模拟多线程工作，每个异步结果都是通过任务队列来调度。队列中存放着一个个task，主要分为两类，分别为macro task 和 micro task，并且每个 macro task 结束后，都要清空所有的 micro task。

在浏览器环境中，

  • 常见的 macro task 有 setTimeout、MessageChannel、postMessage、setImmediate；
  • 常见的 micro task 有 MutationObsever 和 Promise.then。

nextTick就是做这一块的整合。

nextTick的实现在src/core/util/next-tick.js中。

从上面代码可以看到，首先定义两个变量microTimerFunc、macroTimerFunc，分别对应micro task 的函数和 macro task 的函数。

  • 对于macro task，按优先级依次检测setImmediate、MessageChannel是否支持，都不支持就使用setTimeout 0来实现。
  • 对于micro task ，则检测是否支持Promise，不支持则指向macro task的实现，使用macro来代替

从上面flushCallbacks函数可以看到，会遍历callbacks，并且同步执行。说明nextTick会把一个tick内的任务，在一个同步任务内执行完，不会分多次异步去执行。

最以可以看到函数的最后提供了一个Promise化的调用，在不传cb的时候，可以使用promise的方式使用nextTick。

vue提供了两种调nextTick的方式，一种是全局的，上面initGlobalAPI中的定义Vue.nextTick。另一种是实例上的vm.$nextTick,定义在renderMixin()方法中，在vue定义的文件位置。两种方法最后调用的都是上面定义的nextTick方法。

响应式的原理

vue2.x的响应式的核心主要使用了ES5的Object.defineProperty方法。

Object.defineProperty() 的作用是直接在一个对象上定义一个新属性，或者修改一个已经存在的属性

这个方法里的desc属性描述符中包含了get和set以及其它的一些可选键值。

  • get 相当于给对象的这个属性添加了getter方法，访问这个值的时候，会触发getter方法。
  • set相当于给对象的这个属性添加了setter方法，修改这个属性的时候会触发这个方法

这样就相当于将vue中的对象变成响应式了，每次改变都会触发一些方法。

这几个api初始化是在初始化vue的时候eventsMixin(Vue)中，文件定义在src\core\instance\events.js中

vm.$off

从上面可以看到，如果arguments.length === 1,则会执行vm._events[event] = null,相当于移除所有的event。如果提供fn，则会遍历event的数组去splice对应的event数组里的函数。如果传入的事件名是个数组，则会for循环递归调用自身，执行和string相同的操作。

vm.$once

可以看到，使用$once添加事件的时候，会调用$on,并且执行事件的时候会调用$off，移除事件。

从上面可以看到，emit调用的时候，先会去vm._events[event]中查看有无定义这个事件，有的话会去遍历这个event的数组并且执行每一项的方法。从上面off可知，off之后的event会是一个空数组，这里for循环就不会走了。
